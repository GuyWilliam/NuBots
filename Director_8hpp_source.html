80  /**
81  * Compares the priorities of two director tasks and returns true if the challenger has priority over the

83  *
84  * The function requires that the challenger's precedence is strictly greater than the incumbent's.
85  * This ensures that we don't change tasks unnecessarily when the priority is equal.
86  *
87  * @param incumbent the task to compare which is currently the active running task
88  * @param challenger the task to compare which wants to run but is not currently running
89  *
90  * @return true if the challenger has strictly higher priority than the incumbent
91  * @return false if the incumbent task has equal or higher priority
92  *
93  * @throws std::runtime_error if the director's provider ancestry is broken
94  */

109  /**
110  * Reevaluates all of the tasks that are queued to execute on a provider group.
111  *
112  * Each of the tasks in the queue are waiting to use this provider group but cannot for some reason or another.
113  * In order to determine if they can now run on the queue we need to look at each queued task and inspect if the
114  * provider that created them can now run. We do this by simply trying to run them all again.
115  *
116  * @param group the group whose queue we want to reevaluate
117  */
118  void reevaluate_queue(provider::ProviderGroup& group);

120  /**
121  * An object which holds the possible solutions to running a task.
122  * Each solution represents how a single task could be executed. As each provider group can have more than one
123  * provider, there can be more than one option for how to run the task. For each option that we can choose, they
124  * can themselves have their own requirements for provider groups they need to execute. Therefore each of these
125  * requirements is themselves a solution to another task that could be emitted.
126  */
127  struct Solution {
128 
129  /**
130  * A single option for how a provider could be executed
131  */
132  struct Option {
133  /// The state of this provider if it is blocked along with a description of why it is blocked
134  enum State {
135  /// This option is not blocked to us
136  OK,
137  /// This provider is blocked as the task given didn't have enough authority to run it
138  BLOCKED_PRIORITY,
139  /// This provider is blocked due to a when condition that cannot be met
140  BLOCKED_WHEN,
141  /// This provider is blocked as executing it would cause a loop of providers
142  BLOCKED_LOOP,
143  };
144 
145  /// The provider we are executing in this
146  std::shared_ptr<const provider::Provider> provider;
147  /// The set of solutions needed for this option to be executed
148  std::vector<Solution> requirements;
149  /// The state of this option, holds if it is blocked and if so why
150  State state;
151  };
152 
153  /// This value is true if this solution is a pushing solution. Meaning that the main solution cannot run but
154  /// running solution can enable a when condition to be met
155  bool pushed;
156  /// The list of options for this solution
157  std::vector<Option> options;
158  };

160  /**
161  * Return the passed provider as an option for a solution along with its requirements.
162  * It will recursively find Solutions for each of the requirements of the provider including its unmet when
163  * conditions and needs relationships.
164  *
165  * @param provider the provider that we are trying to find a solution for
166  * @param authority the task that we are using as our authority token for permission checks
167  * @param visited the set of providers that have already been visited to prevent loops
168  *
169  * @return
170  */
171  Solution::Option solve_provider(
172  const std::shared_ptr<provider::Provider>& provider,
173  const std::shared_ptr<const ::extension::behaviour::commands::DirectorTask>& authority,
174  std::set<std::shared_ptr<const provider::Provider>> visited);
175 
176  /**
177  * Finds the providers that can cause the passed when condition to be met.
178  * It will then recursively find Solutions for each of the requirements of the provider potentially including
179  * more when conditions that need to be met. The final solution for a task could end up with several steps
180  * removed from the original task.
181  *
182  * @param when the when condition we want to meet by finding causings that will allow a provider to run
183  * @param authority the task that we are using as our authority token for permission checks
184  * @param visited the set of providers that have already been visited to prevent loops
185  *
186  * @return the set of providers that when run can meet the provided when condition
187  */
188  Solution solve_when(const provider::Provider::WhenCondition& when,
189  const std::shared_ptr<const ::extension::behaviour::commands::DirectorTask>& authority,
190  const std::set<std::shared_ptr<const provider::Provider>>& visited);
191 
192  /**
193  * Creates options for each provider in a provider group specified by the passed type.
194  * This function will check each provider in the group and make an option for it.
195  *
196  * @param type the type of task we are trying to run
197  * @param authority the task that we are using as our authority token for permission checks
198  * @param visited the set of providers that have already been visited to prevent loops
199  *
200  * @return the set of possible solution options for the provider group of the passed type
201  */
202  Solution solve_group(const std::type_index& type,
203  const std::shared_ptr<const ::extension::behaviour::commands::DirectorTask>& authority,
204  const std::set<std::shared_ptr<const provider::Provider>>& visited);
205 
206  /**
207  * Finds a solution for a given task. It will start with the provider group for the given task and recursively
208  * work its way down to find all possible solutions. It will use the passed task as an authority token for the
209  * permission checks on providers that it needs to run.
210  *
211  * @param task the task we are finding solutions for
212  * @param authority the task that we are using as our authority token for permission checks
213  *
214  * @return the set of possible solution options for this task
215  */
216  Solution solve_task(const std::shared_ptr<const ::extension::behaviour::commands::DirectorTask>& task,
217  const std::shared_ptr<const ::extension::behaviour::commands::DirectorTask>& authority);
218 
219  /**
220  * Looks at all the tasks that are in the pack and determines if they should run, and if so runs them.
221  *
222  * This function will ensure that if this pack is to be executed it can be executed as a whole. That means that
223  * all of the non optional tasks in it are able to run now given the state of the system. If the system is
224  * unable to run due to priority or due to a when condition not met it will not run any of the tasks in this
225  * pack, but enqueue them on the relevant providers.
226  *
227  * In the event that `When` conditions needs to be met by this pack and it has sufficient priority, it may also
228  * place this provider group into a "Pushing" state whereby it forces another provider group into a state where
229  * it will make the system reach the causing it requires.
230  *
231  * @param pack the task pack that represents the queued tasks
232  */
233  void run_task_pack(const TaskPack& pack);
